Core Features & UX

  • Full CRUD Functionality: Add, edit, complete, and delete tasks seamlessly.
  • Task Organization: Assign priority levels, due dates, and detailed descriptions.
  • Live Filtering: Instantly filter by All, Active, or Completed tasks with updating counters.
  • Data Persistence: Tasks are saved to `localStorage`, making them available across browser sessions.
  • Polished User Experience: Smooth animations, toast notifications for feedback, and delete confirmation modals.

Code Architecture

  • Single Source of Truth: App state is managed in a `tasks` array. The UI is re-rendered from this array after any change.
  • Efficient Event Handling: Using Event Delegation, a single listener on the main list manages all task actions, boosting performance.
  • Direct DOM Manipulation: Pure vanilla JS is used to dynamically render and update the task list from the state array.
  • Local Storage Persistence: The `tasks` array is serialized to JSON and saved to `localStorage` on every change.
  • Modular Functions: Code is organized into clear functions for adding, updating, rendering, and handling tasks.

Security: A Proactive Approach

  • Threat Identified: Cross-Site Scripting (XSS) from user-provided task titles or descriptions.
  • Vulnerability: Injecting user content directly via `.innerHTML` without sanitization.
  • Attack Vector: A payload like <img src=x onerror=alert('hacked')> could execute arbitrary JavaScript.
  • The Fix: Implemented a robust `escapeHtml()` function to neutralize HTML tags, ensuring user input is always treated as plain text before rendering.
